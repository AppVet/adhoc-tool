Install Ad-Hoc Tool Adapter

From the AppVet server, open a browser and download the latest Ad-Hoc tool adapter example file at:

https://github.com/AppVet/adhoc-tool/releases


and move it to your $APPVET_FILES_HOME/conf/tool_adapters/android (or ios) directory.

Next, rename the file with your tool ID which is restricted to lowercase alphabetic characters only (e.g., if your tool is named "My Tool", rename the file to "mytool.xml"). Then edit the XML file to reflect your ad-hoc tool. Note that

should always be set to REPORT, 
 should always be set to NONE,
	and 
AuthenticationRequired
 should always be set to false. 
	
	For more information on
	Tool Adapter Properties, see
	Tool Adapter Properties.
	

	

		
Step 3. Test or Install Tool

	
	
Test


If you are connecting to an existing Androwarn service that is accessible to AppVet, you can test the 
service now by logging into AppVet, uploading an app, and viewing the Androwarn report. If you experience errors 
connecting to the tool, verify the tool adapter properties or contact the tool's host administrator. 

	
Install Tool

If you do not yet have access to an existing Androwarn tool, you must 
install it. For more information, see the Androwarn Service Installation.
